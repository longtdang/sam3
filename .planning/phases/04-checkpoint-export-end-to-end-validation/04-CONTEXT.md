# Phase 4 Context: Checkpoint Export & End-to-End Validation

## Goal
Produce a verified fine-tuned checkpoint on the industrial defect dataset that loads into SAM3 inference scripts without modifications.

## Prior Decisions (from Phases 1–3)
- **D-01**: Data format is CVAT COCO JSON; Phase 1 produced `train.json` / `val.json` split scripts
- **D-02**: Training launcher: `python sam3/train/train.py --config configs/custom_finetune/base --num-gpus N`
- **D-03-02**: Inference API: `sam3.build_sam3_image_model(checkpoint_path=..., enable_segmentation=True)`
- **D-03-06**: `val_epoch_freq: 1` — validates after every epoch
- **D-P3-01**: `save_best_meters` key is already wired in `CheckpointConf` in `trainer.py`

## Phase 4 Decisions

### D-04-01: Dataset path convention
- **Decision**: Real data is NOT yet available. A synthetic fake dataset is generated by the pipeline scripts for CI/smoke-testing.
- **Convention**: When real data becomes available, place it at `data/industrial_defect/` with the following structure:
  ```
  data/industrial_defect/
  ├── train.json          # COCO-format annotations
  ├── val.json            # COCO-format annotations
  └── images/
      ├── train/
      └── val/
  ```
- **Config path key**: `paths.dataset_root` in `base.yaml` — set this to the real data root
- **Smoke test**: A `scripts/generate_fake_dataset.py` script creates a minimal synthetic COCO dataset (5 fake images + masks) at a temp path for dry-run tests

### D-04-02: Best checkpoint filename
- **Decision**: After every epoch where `val_custom/detection` (AP50) improves, the trainer saves `val_custom_detection.pt` to `checkpoint_conf.save_dir`. We will add a post-save hook (or a thin wrapper script) that copies/renames this to `best_checkpoint.pth` in the same directory.
- **Implementation**: A `scripts/rename_best_checkpoint.sh` wrapper is NOT preferred. Instead, subclass or patch the `_save_checkpoint` method in `trainer.py` to also write a `best_checkpoint.pth` alias when a meter in `save_best_meters` is saved.
- **Rationale**: CKPT-01 requires `best_checkpoint.pth` as the canonical output artifact. Downstream users and inference scripts will look for this fixed name.

### D-04-03: Verification script behavior
- **Script**: `scripts/test_checkpoint_compatibility.py`
- **Behavior**: Silent pass — exits 0 on success, non-zero on failure
- **On success**: Logs only: `[OK] Loaded checkpoint: <path>` and `[OK] Output tensor shape: <shape>`
- **Inference call**: Uses `build_sam3_image_model(checkpoint_path=..., enable_segmentation=True)`, then runs one forward pass on a random 1024×1024 tensor input
- **No metrics required**: Does NOT assert AP50 or run a real eval — shape check + no exception = pass (VAL-02)

### D-04-04: Fake dataset format for smoke test
- **Generator script**: `scripts/generate_fake_dataset.py`
- **Output**: Creates a temp dir with 5 synthetic 64×64 PNG images and corresponding COCO JSON annotations with 1 binary mask polygon per image
- **Usage**: `python scripts/generate_fake_dataset.py --out /tmp/fake_defect_data` → prints path for downstream use
- **Category**: Single category `{"id": 1, "name": "defect", "supercategory": "defect"}`

### D-04-05: End-to-end smoke test scope (VAL-01)
- **Decision**: VAL-01 (`AP50 > 0`) is validated using the fake dataset for CI purposes. A real `AP50 > 0` run on real data is a manual step the user performs after setting up `data/industrial_defect/`.
- **CI assertion**: Running training for 1 epoch on the fake dataset must complete without crash (exit 0). AP50 on 5 fake images may be 0 — this is acceptable for CI. The `AP50 > 0` bar applies only to the real dataset.
