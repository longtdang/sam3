Create `sam3/train/configs/custom_finetune/finetune_strategy/full_finetune.yaml` (directory should already exist from Plan 02). Write the following YAML content:

```yaml
# @package _global_
defaults:
  - /configs/custom_finetune/base  # inherit complete config from base.yaml
  - _self_                         # allow fields below to override inherited values

# ============================================================================
# Full Fine-Tuning Strategy (D-09)
#
# Enables training of ALL parameter groups including the ViT vision backbone.
# Recommended for datasets >= 500 images where the backbone can generalize.
# For smaller datasets, use the default decoder-only strategy (base.yaml).
#
# Launch command:
#   python sam3/train/train.py -c custom_finetune/finetune_strategy/full_finetune \
#     --use-cluster 0 --num-gpus 1
#
# Strategy change vs. decoder_only:
#   lr_vision_backbone:  2.5e-6 → 2.5e-5  (10× higher — backbone actively trains)
#   lr_language_backbone: 1.5e-6 → 1.5e-5 (10× higher — language backbone trains)
#   lrd_vision_backbone: 0.9 (same value — now meaningful: deeper layers get lower LR)
# ============================================================================
scratch:
  lr_scale: 1.0                   # full fine-tune: all parameter groups at full learning rate
  lr_vision_backbone: 2.5e-5      # D-09: 10× decoder-only (2.5e-6 → 2.5e-5); enables ViT backbone training
  lr_language_backbone: 1.5e-5    # 10× decoder-only (1.5e-6 → 1.5e-5); enables language backbone training
  lrd_vision_backbone: 0.9        # D-09: layer-wise LR decay across ViT trunk (deeper layers get 0.9^depth × LR)
```

Four overridden fields:
1. `lr_scale: 1.0` — documents full fine-tune mode (informational; LRs are set via explicit literals below)
2. `lr_vision_backbone: 2.5e-5` — 10× higher than decoder-only baseline (2.5e-6); ViT trunk now trains
3. `lr_language_backbone: 1.5e-5` — 10× higher than decoder-only baseline (1.5e-6)
4. `lrd_vision_backbone: 0.9` — same numerical value as base.yaml, but now meaningful because backbone LR is non-trivial

These override the corresponding values in `scratch:` section of base.yaml via Hydra composition. The `trainer.optim.options.lr` schedulers reference `${scratch.lr_vision_backbone}` and `${scratch.lr_language_backbone}` via interpolation — overriding the scratch values automatically propagates to the optimizer configuration.
