# Milestone v1.0: SAM3 Custom Fine-Tuning Pipeline

**Status:** ✅ SHIPPED 2026-05-28
**Phases:** 1–5
**Total Plans:** 13

## Overview

Config-driven fine-tuning pipeline for SAM3 on CVAT COCO-format datasets.
Any team member can point the pipeline at a CVAT COCO export and produce a
fine-tuned SAM3 checkpoint without touching model code.

## Milestone Summary

**Key Decisions:**
| Decision | Rationale |
|----------|-----------|
| `python sam3/train/train.py` (not torchrun) | train.py uses `torch.multiprocessing.start_processes` internally; torchrun would double-spawn workers and crash |
| Sam3Processor public API for inference | Handles BatchedDatapoint construction internally; cleaner than raw model calls |
| SAM3 norms `[0.5, 0.5, 0.5]` | NOT ImageNet; must match Sam3Processor internal normalization |
| best_checkpoint.pth in `{"model": {"detector.<k>": tensor}}` format | HuggingFace inference format satisfies `_load_checkpoint` requirements |
| 4 required config fields (not 3) | `experiment_log_dir` was missing from original spec; corrected in Phase 5 research |
| Freeze ViT backbone by default (decoder_only.yaml) | < 500 images insufficient to update 1B ViT params without overfitting |

**Known Gaps (deferred):**
- Milestone DoD: `Pipeline tested on industrial defect dataset (coco_eval_segm_AP50 > 0)` — requires real GPU training run; deferred post-milestone
- Milestone DoD: `CVAT COCO export → checkpoint in < 30 min of setup` — requires real environment test; deferred post-milestone

**Issues Resolved:**
- torchrun incompatibility discovered during Phase 5 research (D-05-03)
- `experiment_log_dir` missing from original config field list (D-P2-02)
- ImageNet norms mistake corrected to SAM3 `[0.5, 0.5, 0.5]`

**Issues Deferred:**

- Full E2E training validation on real CVAT export — v1.1
